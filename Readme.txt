
# 김영한 스프링 입문 - 코드로 배우는 스프링 부트, 웹 MVC, DB 접근 기술 강의 학습으로 만든 프로젝트

# 중요도
# jdbcTemplate - 반복되는 코드 줄지만, 쿼리 직접 작성
# jpa - 기본적인 crud는 하지만, 쿼리 작성 x ( select 시 jpql )
# springJpa - 구현 클래스 작성 필요 x , 인터페이스로만 구현이 가능함

# Aop
# 공통관심사항 vs 핵심관심사항
# 공통관심사항 분리해줌 ( 핵심 관리 사항'만' 깔끔하게 유지함 )
# method 호출시 interceptor 하여 항상 실행되는 함수
# 서비스 실행시 프록시 스프링 빈을 생성하여( 타켓팅한 주체 ) 먼저 실행시키고 난 뒤, joinPoint.proceed()를 통하여 진짜 스프링 빈을 실행 시켜준다.
